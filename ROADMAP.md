## 🎵 音效规划（按皮肤）

### 经典球皮肤
- 选中："叮"（水晶声）
- 移动：滑动摩擦声
- 消除：爆破音
- 生成：弹跳声

### 马年马皮肤
- 选中：马嘶鸣
- 移动：马蹄声（嗒嗒嗒，速度随距离变化）
- 消除：群马嘶鸣+金币声
- 生成：马匹入场（远处奔来）

### 未来皮肤（预留）
- 科幻：机械音、激光声
- 自然：鸟叫、风声
- 国风：古筝、琵琶

## 🛠️ 技术实现顺序

1. **皮肤架构**（1-2天）
   - 重构 HorseRenderer → 提取通用 SkinRenderer 接口
   - 创建 skins/ 目录结构
   - 实现皮肤加载器

2. **音效系统**（0.5天）
   - Web Audio API 封装
   - 皮肤-音效绑定

3. **设置面板**（0.5天）
   - UI 设计
   - 本地存储

4. **横屏适配**（0.5天）
   - CSS Media Query
   - 动态布局切换

5. **视觉优化**（1-2天）
   - 拟物材质
   - 微交互
   - 特效升级

## ✅ 决策点

1. **先做皮肤架构？**（推荐：是，否则后续都要重写）
2. **音效用 Web Audio 还是 `<audio>` 标签？**（推荐：Web Audio，可控性高）
3. **横屏检测用 CSS 还是 JS？**（推荐：CSS Media Query + JS 辅助）
